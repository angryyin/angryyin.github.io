php垃圾回收机制

Posted on 2019-03-24 | In php

https://www.cnblogs.com/taijun/p/4206770.html
garbage collection
在php5.3版本之前，是没有垃圾回收机智的，只是简单判断一下变脸zval的refcount是否为0，是的话就释放，否则不释放直至进程结束。
其中
refcount:多少个变量是一样的用了相同的值，这个ref就是多少
isref：bool，当refcount大于2的时候，其中一个变量用了地址&的形式进行赋值，那它就变成1了。
1、赋值
<?php

$name = "咖啡色的羊驼";
xdebug_debug_zval('name');

会得到
name:(refcount=1, is_ref=0),string ‘咖啡色的羊驼’ (length=18)
2、增加一个计数
<?php

$name = "咖啡色的羊驼";
$temp_name = $name;
xdebug_debug_zval('name');

会得到
name:(refcount=2, is_ref=0),string ‘咖啡色的羊驼’ (length=18)
3、引用赋值
<?php

$name = "咖啡色的羊驼";
$temp_name = &$name;
xdebug_debug_zval('name');

会得到
name:(refcount=2, is_ref=1),string ‘咖啡色的羊驼’ (length=18)
引用赋值会导致zval通过is_ref来标记是否存在引用的情况
4、数组型的变量
<?php

$name = ['a'=>'咖啡色', 'b'=>'的羊驼'];
xdebug_debug_zval('name');

会得到
name:
(refcount=1, is_ref=0),
array (size=2)
‘a’ => (refcount=1, is_ref=0),string ‘咖啡色’ (length=9)
‘b’ => (refcount=1, is_ref=0),string ‘的羊驼’ (length=9)
对于数组来看是一个整体，对于内部kv来看又是分别独立的整体，各自都维护着一套zval的refount和is_ref。

5、销毁变量
<?php

$name = "咖啡色的羊驼";
$temp_name = $name;
xdebug_debug_zval('name');
unset($temp_name);
xdebug_debug_zval('name');

会出现
name:(refcount=2, is_ref=0),string ‘咖啡色的羊驼’ (length=18)
name:(refcount=1, is_ref=0),string ‘咖啡色的羊驼’ (length=18)
refcount计数减1，说明unset并非一定会释放内存，当有两个变量指向的时候，并非会释放变量占用的内存，只是refcount减1.

从内存变化上来看
$name = “咖啡色的羊驼”;
内存的分配做了两件事情：1.为变量名分配内存，存入符号表 2.为变量值分配内存
